Asteroid Tracker - NEO Watch

Why create this project ?

Interest in space and scientific engineering.

This project was born out of my passion for the engineering behind space observation tools. It stems from a desire to understand, model, represent, and make accessible real astronomical phenomena, particularly near-Earth objects (NEOs).

Preparing for NASA Space Apps Challenge Hackathon 2025 :

This project is specifically preparing me for a NASA hackathon, with challenges in data science, modeling, visualization, and innovation. The goal is to gain experience, rigor, understanding of spatial datasets, mastery of scientific APIs, and the ability to quickly produce reliable prototypes.

Reference to NEO Surveyor.

This project is a scaled-down version of the NEO Surveyor mission. NEO Surveyor is the first space telescope designed specifically to detect potentially hazardous asteroids and comets.

Language / Technologies :

Python, Streamlit, Pandas, Matplotid, API NASA NeoWs

Issues & Objectives

Scientific issues

How to effectively track near-Earth asteroids, identify the most dangerous ones based on their size and trajectory, and visualize this information to raise awareness or issue alerts.

Functional objectives
Query NASA datasets (NeoWs API) to obtain asteroid flybys.
Allow selection of a date range for the search.
Display results in an interactive table (name, estimated diameter, date of passage, hazard status).
Generate visualizations (scatter plots, etc.) showing the correlation between size, date, and dangerous status.

Methodology

Stage Description
Data acquisition Query NASA datasets (NeoWs API) to obtain asteroid flybys.
Pre-treatment / Cleaning Transformation of raw data (dates, dimensions, hazardous status), format management, handling of missing values.
Analysis / Classification Sorting and filtering by diameter, date, hazardous status; preparing data for visualization.
Visualization & Interface Web interface via Streamlit, graphs with Matplotlib, interactive tables.
Validation Comparison with official NASA data to ensure reliability.

Visualization

This graph shows the distribution of detected asteroids based on their diameter (Y axis) and date of passage (X axis). The red dots represent objects classified as potentially hazardous, while the green dots indicate harmless objects. This representation allows us to quickly identify days when several large asteroids pass close to Earth and prioritize observations.

After selecting a desired observation period, the application performs a request to the NASA NeoWs (Near Earth Object Web Service) API. The API returns a response containing, for each detected asteroid :

  • The close-approach date
  • The asteroidâ€™s name
  • Its estimated diameter (in meters)
  • A flag indicating whether it is classified as potentially hazardous

This data is then processed and rendered in a dynamic table, allowing for quick and intuitive analysis.

Asteroids identified as passing close to Earth are classified in a table listing :

  • Their close-approach date
  • Their name
